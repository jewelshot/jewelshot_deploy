# Accessibility Statement

## Our Commitment

Jewelshot is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

## Conformance Status

We aim to conform to the **Web Content Accessibility Guidelines (WCAG) 2.1 level AA**. These guidelines explain how to make web content more accessible for people with disabilities and user-friendly for everyone.

## Measures to Support Accessibility

Jewelshot takes the following measures to ensure accessibility:

- Integrate accessibility into our development processes
- Employ formal accessibility quality assurance methods
- Maintain keyboard navigation for all functionality
- Provide text alternatives for non-text content
- Ensure sufficient color contrast throughout the interface
- Support screen readers and assistive technologies

## Features Implemented

### Keyboard Navigation
- ✅ All interactive elements accessible via keyboard
- ✅ Logical tab order throughout the application
- ✅ Visible focus indicators for keyboard users
- ✅ Keyboard shortcuts for common actions (documented in-app with `?` key)
- ✅ Skip links to bypass repetitive navigation

### Screen Reader Support
- ✅ ARIA labels on all interactive elements
- ✅ Semantic HTML landmarks (main, nav, complementary)
- ✅ Alternative text for all meaningful images
- ✅ ARIA live regions for dynamic content
- ✅ Proper heading hierarchy

### Visual Design
- ✅ Minimum 4.5:1 color contrast ratio for normal text
- ✅ Minimum 3:1 color contrast ratio for large text
- ✅ Focus indicators visible against all backgrounds
- ✅ Text resizable up to 200% without loss of functionality
- ✅ Support for high contrast mode

### Forms and Input
- ✅ Labels associated with all form controls
- ✅ Error messages clearly identified
- ✅ Required fields marked appropriately
- ✅ Validation feedback provided

### Multimedia
- ✅ Alternative text for images
- ✅ Reduced motion support for animations
- ✅ No auto-playing audio or video

## Keyboard Shortcuts

Press `?` anywhere in the application to view all available keyboard shortcuts.

Common shortcuts:
- **Tab**: Navigate forward through interactive elements
- **Shift + Tab**: Navigate backward
- **Enter/Space**: Activate buttons and links
- **Escape**: Close modals and panels
- **Ctrl/Cmd + O**: Open file upload (Studio)
- **Ctrl/Cmd + S**: Save image (Studio)
- **+/-**: Zoom in/out (Studio)
- **0**: Fit to screen (Studio)

## Known Issues

We are aware of the following accessibility limitations:

1. Some AI-generated images may lack descriptive alternative text (we encourage users to add custom descriptions)
2. Color picker tools may be challenging for users with color blindness (we're working on improvements)
3. Some advanced canvas manipulations may be difficult without visual feedback

## Compatibility

Jewelshot is designed to be compatible with the following assistive technologies:

- **Screen Readers**: NVDA, JAWS, VoiceOver (macOS/iOS), TalkBack (Android)
- **Browser Extensions**: High contrast mode, screen magnifiers
- **Keyboard-only navigation**: Full support across all features
- **Voice control**: Works with Dragon NaturallySpeaking and similar tools

### Supported Browsers

- Chrome (latest)
- Firefox (latest)
- Safari (latest)
- Edge (latest)

Jewelshot's accessibility relies on the following technologies:

- HTML5
- WAI-ARIA 1.2
- CSS3
- JavaScript (required for full functionality)
- Next.js 14 (React framework)

## Assessment Approach

Jewelshot assesses accessibility through:

- Self-evaluation using automated tools (Lighthouse, axe DevTools)
- Manual keyboard navigation testing
- Screen reader testing (VoiceOver, NVDA)
- Color contrast analysis (WebAIM Contrast Checker)
- Code reviews for WCAG compliance

If you are not satisfied with our response to your accessibility feedback, you may file a formal complaint. We are committed to resolving accessibility issues promptly.
